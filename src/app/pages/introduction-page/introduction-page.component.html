Markup, Javascript, Pure CSS

Srcry differs from other frameworks by providing a variety of dynamic sizes and new CSS properties that give developers more control over the scaling behavior of their elements. This means there's no components or boilerplate code to copy and paste, just values and properties for building your own to perfection. These properties and values are organized into 3 size libraries and 2 configurable classes that make responsive design easier than ever.

What Can We Do With Srcry?

Create A Single Ruleset For All Devices

When transitioning between mobile and desktop screens we run into a few quirks. While things like pictures and titles need to shrink on mobile devices so they don't dominate the screen, things like links and buttons need to grow larger so the user can interact with them using their thumbs and fingers as opposed to a mouse arrow. Normally we use media queries to address this problem but Srcry handles this right out of the box. Scale the window bellow to see this in action.

As you can see when you shrink the screen to around 880px the nav bar starts to grow taller as the screen gets narrower. The only media query used in this example is to toggle the burger icon on and off.

Rapid Scaling When You Need It

Sometimes we want an element to subtly adapt to the screen as it grows rather than staying the same size until the next media query. We can easily achieve this using em, rem and view units but we often run into an issue with the element growing too large before the next break point. If we can't find the magical value needed for it to scale properly we'll just add another media query at the screen size it starts to grow out of control and knock it back to where we want it so it can transition smoothly to the next breakpoint. We don't need to worry about that anymore because Srcry allows you to fine tune this type of scaling within a single set of rules as shown bellow.

In this example the product component adapts subtly as you expand the screen then around 490px it starts growing rapidly. At around 620px the rapid growth stops and it goes back to adapting subtly.

Fluid Proportional Text

Text heirarchy is one of the most important aspects of good web design as it helps the user understand how the content is organized. A lot of us use em units to maintain a proportional difference with our text. This works great at one range of screen sizes however we don't have to scale too far before we need to adjust that proportional difference to keep the text readable and balanced. The following example shows how Srcry addresses this challenge thus eliminating another media query or two.

Starting on the smaller screen all the text is pretty close in size with just enough of a proportional difference to distinguish them from one another. As you enlarged the screen the proportional difference between each type of text became more drastic.
