Markup, Javascript, Pure CSS

Srcry differs from other frameworks by providing a variety of dynamic sizes and new CSS properties that give developers more control over the scaling behavior of their elements. This means there's no component libraries or boilerplate code to modify, just values and properties for building your own elements to perfection. These properties and values are organized into 3 size libraries and 2 configurable classes that make responsive design easier than ever.

What Can We Do With Srcry?

Create A Single Ruleset For All Devices

When transitioning between mobile and desktop screens we run into a few quirks. While things like pictures and titles need to shrink on mobile devices so they don't dominate the screen, things like links and buttons need to grow larger so the user can interact with them using their thumbs and fingers. Normally we use media queries to address this problem but Srcry handles this right out of the box. Resize the window below to see this in action.

As you just saw, when you shrink the screen to around 880px the nav bar and login button grows taller as the screen gets narrower. The only media query used in this example is to toggle the burger icon on and off.

Switch Between Subtle and Rapid Scaling As Needed

Sometimes we want an element to subtly adapt to the screen as it grows rather than staying the same size until the next media query. We can easily achieve this using em, rem and view units but we often run into an issue with the element growing too large before the next break point. If we can't find the magical value needed for it to scale properly we'll just add another media query at the screen size it starts to grow out of control and knock it back to where we want it so it can transition smoothly to the next breakpoint. Srcry eliminates that headache by allowing you to fine tune this type of scaling within a single set of rules as demonstrated in the next example.

In this example, the product component adapts subtly as you expand the screen until around 490px where it starts growing rapidly. At around 620px the rapid growth stops and it goes back to adapting subtly.

Fluid Proportional Text

Text hierarchy is one of the most important aspects of good web design as it helps the user understand at a glance how the content is organized. A lot of us use em units to maintain a proportional difference with our text which works great however we don't have to scale too far before we run into problems with runaway text. Starting out, all the text is fairly close in size with just enough of a proportional difference to distinguish them from one another. As you enlarge the screen, the proportions become greater making good use of the extra white space that becomes available.
