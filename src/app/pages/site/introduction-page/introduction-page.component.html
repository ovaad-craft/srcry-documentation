What You Can Do With Srcry

When transitioning between mobile and desktop screens we run into a few quirks. While things like pictures and titles need to shrink on mobile devices so they don't dominate the screen, things like links and buttons need to grow larger so the user can interact with them using their thumbs and fingers as opposed to a mouse arrow. Normally we use media queries to address this problem but Srcry handles this right out of the box. Scale the window bellow to see this in action.

As you can see when you shrink the screen to around 880px the nav bar starts to grow taller as the screen gets narrower. The only media query used in this example is to toggle the burger icon on and off.

Sometimes we want an element to subtly adapt to the screen as it grows rather than staying the same size until the next media query. We can easily achieve this using em, rem and view units but we often run into an issue with the element growing too large before the next break point. If we can't find the magical value needed for it to scale properly we'll just add another media query at the screen size it starts to grow out of control and knock it back to where we want it so it can transition smoothly to the next breakpoint.
